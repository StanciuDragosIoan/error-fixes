Fix error cli.init is not a function when initializing react native cli project

In order to fix this error, you simply pass and option to the command, in order to 'downgrade' the cli version (there's some issue with the current version) such as:
npx react-native init myAwesomeApp --version 0.68.2

Fix error SDK location not found. Define location with an ANDROID_SDK_ROOT environment variable or by setting the sdk.dir path in your project's local.properties file when running react native cli project

In order to fix this error, you need to create a local.properties file inside the 'android' directory and paste inside:
sdk.dir = /home/YOUR_USER/Android/Sdk

All React Native CLI variables to add in .bashrc

export ANDROID_SDK_ROOT=$HOME/Android/Sdk export ANDROID_HOME=$HOME/Android/Sdk export PATH=$PATH:$ANDROID_HOME/emulator:$PATH export PATH=$PATH:$ANDROID_HOME/tools:$PATH export PATH=$PATH:$ANDROID_HOME/tools/bin:$PATH export PATH=$PATH:$ANDROID_HOME/platform-tools:$PATH

Kill adb server

adb kill-server
